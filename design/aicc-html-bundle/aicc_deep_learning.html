01 — 类比天才外科医生 · 分诊台

一家顶级医院有位天才外科医生，能做最复杂的手术，也能用同样的技艺造成巨大伤害。过去，安全与能力始终像是鱼与熊掌——

旧路 A把天才锁进办公室 —— 安全了，能力也废了。
旧路 B塞一本厚厚的禁令手册 —— 他畏首畏尾，正常学术问题也拒答。
换一种思路
门口加一位分诊护士

常规感冒发烧，护士直接放行，天才医生全力服务；可一旦听到"如何制造炸弹"，她不让天才出面、也不生硬拒绝，而是微笑引导到隔壁那位专攻常规、不具高危能力的资深医生。顶尖能力照常开放，高危风险悄然化解，病人也不觉得被冒犯。

02 — 技术定义运行时安全架构

把安全策略从模型权重中解耦，做成一个独立的外部执行层。能力照常发挥，安全独立兜底。

Fable 5 与 Mythos 5 共享底层模型，前者外挂分类器面向公众 · 切换对用户透明，并告知"本次由备用模型生成"

本质，是把"能力边界"从静态训练阶段，移到动态运行时——大模型安全由此从提示词工程问题，升级为一项系统工程：

能力分层场景识别身份权限审计日志合规策略
拆开来看 · 三个零件
通俗类比安全分类器
像机场安检的 X 光机，不看你是谁，只看这件行李是否属于高风险类别，快速判别并放行或拦截。
技术视角：一组独立训练的模型，专门对用户输入做实时风险判定，输出风险标签与置信度，作为路由决策依据，无需重训庞大基座模型。
通俗类比静默降级路由
像网银里的风控：当系统判定有风险，会把你切到更严格的复核通道，你几乎无感，但安全级别已悄悄提上去。
技术视角：在分类器触发时把请求交给备用模型，对用户透明、交互连贯，避免生硬拒绝，体验顺滑。
通俗类比模型蒸馏
像偷师学艺：用顶尖大师的作品训练自己的学徒，让学徒快速逼近大师水平，而不用从头培养。
技术视角：用高能力"教师模型"的输出训练更小、更高效的"学生模型"。护栏要防的，正是有人借此复制核心能力。

分步静态帧 · 当前技术实现原理

桌面计算机使用本质上是一个基于视觉的马尔可夫决策过程：模型在每一步截取屏幕图像作为状态，通过多模态编码器理解界面，再经策略网络生成鼠标键盘动作，执行后进入下一个状态，循环直至任务完成。

1
屏幕状态捕获与视觉编码s_t → h_t = VisionEncoder(s_t)
操作系统 API 截取当前窗口截图转为像素矩阵，送入多模态编码器，识别按钮、文本框、图标等 UI 元素及其空间关系——相当于给 AI 注入视觉感知，从"盲人"变"明眼人"。
2
任务状态理解与上下文融合z_t = CrossAttention(h_t, task)
视觉特征与任务指令跨模态融合，把"复制第三列到邮件"与屏幕 UI 对齐，理解哪个按钮是"复制"、哪块区域是"第三列"，建立意图与像素位置的映射。
3
动作空间生成与策略推理a_t = PolicyNetwork(z_t)
基于融合状态，策略网络同时生成动作类型（移动 / 单击 / 双击 / 输入 / 滚动 / 拖拽 / 等待）与动作参数（目标坐标、输入字符串、滚动距离）——核心决策环节。
4
动作执行与环境反馈Execute(a_t) → s_t+1
通过系统级输入接口执行（Windows SendInput / macOS CGEvent），等待界面响应后进入下一轮，重新截屏观察新状态。
5
循环终止与任务完成判定ifDone(z_t) → Report
每轮额外输出终止信号，判断屏幕状态是否匹配目标终态、是否出现错误弹窗、子步骤是否完成。未完成则回到第 1 步，完成则向用户返回结果摘要。

关键点：这张图的核心是右侧那条虚线回路，它揭示了桌面控制最本质的机制——观察 · 行动 · 再观察的闭环。AI 不是一次性输出所有操作，而是每做一步就重看一眼屏幕，根据最新状态调整下一步。
