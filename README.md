## What is AI-33CC?

**AI-33CC** (AI Concept Coach) is a **Feynman-based penetration learning platform** built around the "3+4=1" coaching framework:

- **3 Warm-up Questions** (dynamically generated by LLM from 3 roles: Business Director / CTO / Developer)
- **4 Learning Steps** (Intuition -> Scenario -> Mechanism -> Essence)
- **= 1 AI Concept Mastered** in under 15 minutes

Designed for non-algorithm practitioners in the MaaS industry who need to explain complex AI concepts (GDN, LoRA, Ring Attention, MoE, Flash Attention, etc.) to clients with confidence. Ships with a **built-in knowledge base of 30 hot AI/LLM algorithm concepts** (2022+) with arXiv paper links.

> **Why "3+4=1"?**  
> Research shows that predicting answers before learning (3 warm-up questions) reduces the illusion of competence by 40%. Combined with progressive 4-step deepening, users achieve true conceptual mastery rather than surface-level familiarity — hence "1 concept mastered" in one session.

## Why AICC?

| Pain Point | How AICC Solves It |
|------------|-------------------|
| **Cognitive gap**: Papers only explain math **or** business, not both | **Four-step progressive flow**: Intuition -> Scenario -> Mechanism -> Essence |
| **High cognitive load**: Abstract terms, dense formulas, hard to digest | **Feynman Method**: 3 warm-up questions + role-based explanations + analogies + dynamic SVG |
| **Knowledge fragmentation**: Chatbox answers are lost after one session | **Local knowledge graph** auto-mounting + note persistence + multi-format export |

## Key Features

### 1. The "3+4=1" Coaching Framework

#### 3 Warm-up Questions (Feynman Pre-heat)
- LLM dynamically generates **3 role-based questions** (Business Director / CTO / Developer)
- Progressive card reveal to reduce perceived wait time
- User-triggered: waits for explicit confirmation before starting (no auto-advance)

#### 4 Learning Steps (Progressive Deepening)
| Step | Focus | Output |
|------|-------|--------|
| **Step 1: Intuition** | "What is it?" | Analogy + Definition + Glossary + Takeaway |
| **Step 2: Scenario** | "Where to use?" | Applicable/Inapplicable scenarios + Criteria |
| **Step 3: Mechanism** | "How does it work?" | Step-by-step frames + **Dynamic SVG** + Math |
| **Step 4: Essence** | "One sentence?" | One-liner + Anchor + Contrast + 3 Takeaways |

#### = 1 Concept Mastered
- Each step confirms understanding before advancing
- Auto-mounts to local knowledge graph for long-term retention

### 2. Dynamic SVG Mechanism Diagrams
- **LLM-generated SVG** in Step 3: each algorithm gets a unique diagram reflecting its core mechanism
- No more generic "Input -> Transform -> Output" fallback
- SVG spec enforced via prompt: `viewBox="0 0 600 320"`, labeled nodes, directional arrows, highlighted key components
- Examples: LoRA shows frozen W + low-rank bypass; Ring Attention shows GPU ring topology; MoE shows router -> expert selection

### 3. 30 Built-in Algorithm Concepts
- Curated knowledge base of hot AI/LLM concepts since 2022
- Categories: Attention, Position Encoding, Architecture, Training/Alignment, Efficient Fine-tuning, Inference Acceleration, Reasoning Paradigms
- Each with arXiv paper links, randomly sampled as homepage examples
- Source: [`src/feynman/data/algorithm-concepts.ts`](src/feynman/data/algorithm-concepts.ts)

### 4. Four-Step Streaming Pipeline
- **Step 1 (Intuition)**: Real-world analogy, authoritative definition, glossary breakdown
- **Step 2 (Scenario)**: Business applicability analysis with fit ratings
- **Step 3 (Mechanism)**: Step-by-step principle + **dynamic SVG diagram** + token-level math (LaTeX)
- **Step 4 (Essence)**: McKinsey-style one-liner + anchor metaphor + before/after contrast

### 5. Progressive Rendering with Skeleton States
- Each step has **segmented skeleton placeholders** with role-specific loading narratives
- No black JSON boxes during streaming — users see meaningful progress at every stage
- Shared `StreamingSection` component ensures consistent UX across all steps

### 6. Local-First Architecture
- **No backend required**: All LLM calls go directly to DashScope (Alibaba Cloud Bailian)
- **Offline mock mode**: Pre-recorded fixtures for rapid UI debugging
- **Note persistence**: localStorage with Markdown export
- **Knowledge graph**: Visualization of learned concepts

## Getting Started

### Prerequisites
- Node.js >= 18
- Alibaba Cloud DashScope API Key ([Get Key](https://help.aliyun.com/zh/dashscope/developer-reference/api-key-management))

### Installation

```bash
# Clone
git clone https://github.com/huangyong0574/AI-33CC.git
cd AI-33CC

# Install
npm install

# Run dev server (fixed port 5180)
npm run dev

# Build for production
npm run build
npm run preview
```

### Configuration

1. Open `http://localhost:5180` in browser
2. Click **Settings** (gear icon) -> enter your DashScope API Key
3. Toggle **Offline Mock** for offline development (uses pre-recorded fixtures)
4. Default model: `deepseek-v4-flash` (configurable)

## LLM Integration

### API Calls Matrix

| Endpoint | Function | Model | Streaming | Web Search | Thinking |
|----------|----------|-------|-----------|-----------|----------|
| Feynman Warm-up | `callFeynmanWarmup()` | deepseek-v4-flash | No | Yes | No |
| Step 1 | `callStep("step1")` | deepseek-v4-flash | Yes | Yes | Yes |
| Step 2 | `callStep("step2")` | deepseek-v4-flash | Yes | Yes | Yes |
| Step 3 | `callStep("step3")` | deepseek-v4-flash | Yes | No | Yes |
| Step 4 | `callStep("step4")` | deepseek-v4-flash | Yes | No | Yes |

### Prompt Engineering

All prompts are centralized in [`src/feynman/lib/prompts.ts`](src/feynman/lib/prompts.ts):
- **`SYSTEM_STEP1..4`**: Independent system prompts per step
- **`BUILD_STEP_PROMPT`**: Step-specific schema + guidance
- **SVG generation spec**: Embedded in Step 3 prompt for dynamic mechanism diagrams
- **JSON Schema enforcement**: Every step output is validated against strict JSON schema

## Design Philosophy

### Progressive Information Layering

1. **Step 1**: "What is it?" — intuitive analogy + definition (no math, no code)
2. **Step 2**: "Where to use?" — business scenario analysis (no algorithm details)
3. **Step 3**: "How does it work?" — algorithm breakdown + dynamic SVG + math walkthrough
4. **Step 4**: "One sentence?" — McKinsey-style essence distillation

Each step builds on the previous, **never repeating** content. The dynamic SVG in Step 3 is the single source of truth for mechanism visualization — each algorithm gets a unique diagram.

### Feynman Method Integration

- **Warm-up**: 3 questions force users to predict answers before learning (reduces illusion of competence)
- **Takeaway cards**: After each step, users "collect" key insights via modal interaction
- **Progressive depth**: L1 (anyone) -> L2 (PM) -> L3 (engineer) -> L4 (expert)
