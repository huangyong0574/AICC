## What is AI-33CC?

**AI-33CC** (AI Concept Coach) is a **Feynman-based penetration learning platform** built around the "3+3=1" coaching framework:

- **3 Warm-up Questions** (dynamically generated by LLM from 3 roles: Business Director / CTO / Developer)
- **3 Learning Steps** (Intuition → Algorithm → Commercial Value)
- **= 1 AI Concept Mastered** in under 15 minutes

Designed for non-algorithm practitioners in the MaaS industry who need to explain complex AI concepts (GDN, Attention, Mamba, MoE, etc.) to clients with confidence.

> **Why "3+3=1"?**  
> Research shows that predicting answers before learning (3 warm-up questions) reduces the illusion of competence by 40%. Combined with progressive 3-step deepening, users achieve true conceptual mastery rather than surface-level familiarity — hence "1 concept mastered" in one session.

## Key Features

### 🎯 1. The "3+3=1" Coaching Framework

#### 3 Warm-up Questions (Feynman Pre-heat)
- LLM dynamically generates **3 role-based questions** (Business Director / CTO / Developer)
- Progressive card reveal to reduce perceived wait time
- User-triggered: waits for explicit confirmation before starting (no auto-advance)

#### 3 Learning Steps (Progressive Deepening)
| Step | Focus | Output | Time |
|------|-------|--------|------|
| **Step 1: Intuition** | "Let's paint a picture first" | Analogy + Definition + Glossary + SVG sketch | ~4 min |
| **Step 2: Algorithm** | "Now see the mechanism" | Timeline + Animation + Math walkthrough (LaTeX) | ~6 min |
| **Step 3: Value** | "What does this mean for business?" | Engineering metrics + ROI scenarios | ~4 min |

#### = 1 Concept Mastered
- User fills answers from 3 perspectives in Feynman Internalization Panel
- LLM evaluates responses, identifies gaps, suggests next direction
- Auto-mounts to local knowledge graph for long-term retention

### 📖 2. Three-Step Streaming Pipeline
- **Step 1 (Intuition)**: Real-world analogy, authoritative definition, glossary breakdown, **LLM-generated SVG concept sketch**
- **Step 2 (Algorithm)**: Tech evolution timeline, step-by-step animated mechanism, real token math walkthrough (LaTeX)
- **Step 3 (Value)**: Engineering metrics comparison, business ROI scenarios, MaaS deployment analysis

### 🧠 3. Progressive Rendering with Skeleton States
- Each step has **segmented skeleton placeholders** with role-specific loading narratives
- No black JSON boxes during streaming — users see meaningful progress at every stage
- Shared `StreamingSection` component ensures consistent UX across all steps

### 📊 4. Feynman Internalization & Knowledge Graph
- User fills in answers from 3 perspectives (biz/cto/dev)
- LLM evaluates responses, provides gap analysis, suggests next learning direction
- Auto-mounts to local knowledge graph (localStorage), builds long-term cognitive connections

###  5. Local-First Architecture
- **No backend required**: All LLM calls go directly to DashScope (Alibaba Cloud Bailian)
- **Offline mock mode**: Pre-recorded fixtures for rapid UI debugging
- **Note persistence**: localStorage with Markdown/PDF export
- **Knowledge graph**: Interactive D3.js force-directed visualization of learned concepts

## Getting Started

### Prerequisites
- Node.js ≥ 18
- Alibaba Cloud DashScope API Key ([Get Key](https://help.aliyun.com/zh/dashscope/developer-reference/api-key-management))

### Installation

```bash
# Clone
git clone https://github.com/huangyong0574/AI-33CC.git
cd AI-33CC

# Install
npm install

# Run dev server
npm run dev

# Build for production
npm run build
npm run preview
```

### Configuration

1. Open `http://localhost:5173` in browser
2. Click **Settings** (gear icon) → enter your DashScope API Key
3. Toggle **Offline Mock** for offline development (uses pre-recorded GDN fixtures)
4. Default model: `qwen3.6-plus` (configurable)

## Design Philosophy

### Progressive Information Layering

1. **Step 1**: "Let's paint a picture first" — intuitive analogy + definition (no math, no code)
2. **Step 2**: "Now let's see the mechanism" — algorithm breakdown + math walkthrough (animation is **exclusive** to this step)
3. **Step 3**: "What does this mean for business?" — engineering metrics + ROI analysis (no more theory)

Each step builds on the previous, **never repeating** content. The SVG sketch in Step 1 is purely visual; the animation in Step 2 is the single source of truth for mechanism explanation.

### Feynman Method Integration

- **Warm-up**: 3 questions force users to predict answers before learning (reduces illusion of competence)
- **Loop questions**: After each step, users self-test understanding
- **Internalization**: Final review from 3 perspectives (biz/cto/dev) ensures multi-dimensional comprehension
