## Why

当前选题约稿是「一个知识点 → 套话模板」（`从「X」出发，写一篇讲得透的深度文`），缺张力、也**不对准受众**。AICC 用户多是售前 / AI 业务架构师 / 技术销售，他们的读者是**正在做 AI Native 组织转型的客户**。选题应在**融合多个已闭环知识点 × 行业趋势**的基础上，**命中转型客户的决策关切**（战略 / 组织 / 能力 / 落地 / 治理），让用户写出客户真正想读、且能引发讨论的深度文——而不是泛泛的技术科普。

## What Changes

- 选题改为 **LLM 生成**：输入用户**已闭环知识点集合 + 本周雷达趋势/news**，输出**融合 / 对比型**选题，且**每条都对准「AI Native 组织转型客户」的关切**。
- 每个选题卡新增：**角度标签**（面向转型客户的决策视角：战略抉择 / 组织变革 / 能力跃迁 / 落地治理 / 趋势预判）、**客户共鸣度**（1–5★）、**「为什么现在」行业钩子**、**引用的多个闭环知识点** chips。
- **换一批**：重新生成一组选题。
- **门槛（宁缺毋滥）**：闭环知识点不足时锁定并提示「再闭环 N 个解锁」。
- 选题结果**缓存**到 localStorage，避免每次进页面都调用 LLM。
- 不变：AI 只给角度 / 时机 / 素材，**文章仍由用户自己写**。

## Capabilities

### New Capabilities
- `creation`: 创作模块行为契约。本变更落「选题约稿——多知识点 × 趋势融合、角度化、有张力」子集（与 `creation-writing-desk` 同属 `creation`，互不重叠：本变更管**选题生成**，那个管**写作台**）。

## Impact

- 代码：`src/pages/CreationPage.tsx`（`Topic` 类型扩展 + `TopicsView` / `TopicCard` 重做 + 「换一批」）；新增 LLM 调用（复用 `src/feynman/lib/llm.ts` + `aicc-llm-cfg`）。
- 存储契约：新增 `aicc-creation-topics`（选题缓存）。
- LLM：新增「选题生成」调用点 → SPEC §4.4 LLM 调用矩阵补一行（同步 SPEC.md）。
- 风险：选题张力 / 融合是否牵强须真 key 真机验证（gate）。
